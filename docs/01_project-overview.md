# プロジェクト概要書

## 1. プロジェクト背景

DX推進業務において、PJ案件として管理されている業務以外にも、保守作業や突発的な依頼対応など多くの作業が日常的に発生している。しかし、現状ではこれらの作業を把握する仕組みがなく、誰が何をやっているのかがチーム全体で見えない状態にある。

案件外作業には以下の3種類がある。

- **定常保守**: サーバーアップデート、ネットワーク点検、各部屋の点検など定期的に発生する保守作業
- **突発保守**: ネットワーク障害、PowerApps対応など緊急性のある保守対応
- **依頼対応**: 保守ではなく、他部署やメンバーからの突発的な依頼事項

これらが案件化されずに実施されているため、チーム全体での状況把握が困難になっている。

## 2. 目的

本PJの目的は、**案件化されている業務以外に実施している作業の内容を把握する仕組み**を構築することである。

まずは運用を回してみて、どんな問題があるのかを可視化していく。

## 3. 方針

- **MVPとして**、案件化されている業務以外に「誰が」「何をやっているのか」を管理する
- MVP運用が定着した後、実際の**作業時間の管理**へ段階的に拡張する

## 4. 利用シーン

- **毎週の定例会**において、前回定例会から今回までに実施した案件外作業を報告するツールとして使用する
- 各メンバーが作業を実施した際に都度登録し、定例会の場でチーム全体で振り返る

## 5. 対象ユーザー

- PJメンバー全員
- 各メンバーが自分の作業を各自で登録する（代理登録は不要）

## 6. MVPスコープ

### 含めるもの

- 案件外作業の登録・一覧表示
- 作業の分類（定常保守 / 突発保守 / 依頼対応）
- 作業種別（問い合わせ対応、バグ修正、定期メンテナンス、ドキュメント更新、その他）
- 担当者の自動記録（ログインユーザー）
- 期間フィルタ（今日 / 今週 / 今月）による絞り込み
- 分類別・作業種別でのグルーピング表示

### 含めないもの（将来対応）

- 作業時間の記録・管理
- ダッシュボード・集計機能
- 通知機能

## 7. 技術方針

まずはモックを作成し、チーム内でUIや運用フローを検証する。

- **技術スタック**: HTML + Tailwind CSS + JavaScript（単一HTMLファイル）
- **データ保存**: localStorage（モック段階）
- **方針**: 静的モックから開始し、運用のフィードバックを得てから本実装の技術選定を行う
