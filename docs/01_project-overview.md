# プロジェクト概要書

## 1. プロジェクト背景

DX推進業務において、PJ案件とは別に保守作業（問い合わせ対応、バグ修正、定期メンテナンス、ドキュメント更新など）を8名のメンバーで担っている。しかし、現状では誰がどの作業を行っているかを把握する仕組みがなく、各メンバーの作業内容や負荷が見えない状態にある。

保守作業には定常業務（サーバーアップデート、ネットワーク点検、各部屋の点検など）と突発業務（ネットワーク障害、PowerApps対応、突発依頼など）があり、これらが案件化されずに実施されているため、チーム全体での状況把握が困難になっている。

## 2. 目的・ゴール

案件化されていない保守作業を可視化し、誰がどんな作業をしているかをチーム全体で把握できるようにする。

- **短期ゴール（MVP）**: 保守作業の一覧表示による可視化
- **中長期ゴール**: 各作業にかかる時間の記録・管理による工数の見える化

## 3. 対象ユーザーと利用シーン

- **対象ユーザー**: PJメンバー全員（8名）
- **利用シーン**: 案件化されていない保守作業を実施する際に、各自が作業を登録・確認する
- **登録方法**: 各メンバーが自分の作業を各自で登録

## 4. MVPスコープ

### 含めるもの

- 保守作業の登録・一覧表示
- 作業の分類（定常業務 / 突発業務）
- 担当者の表示
- 作業種別（問い合わせ対応、バグ修正、定期メンテナンス、ドキュメント更新など）

### 含めないもの（将来対応）

- 作業時間の記録・管理
- ダッシュボード・集計機能
- 通知機能

## 5. 技術方針

まずはモックを作成し、チーム内でUIや運用フローを検証する。

- **技術スタック**: HTML + Tailwind CSS + JavaScript
- **方針**: 静的モックから開始し、フィードバックを得てから本実装の技術選定を行う
