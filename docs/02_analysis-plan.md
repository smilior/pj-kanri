# データ分析計画書

## 1. 分析の前提

本ツールで蓄積されるデータは以下の項目である。

| データ項目 | 内容 |
|-----------|------|
| 作業名 | 実施した作業の概要 |
| 分類 | 定常保守 / 突発保守 / 依頼対応 |
| 作業種別 | 問い合わせ対応 / バグ修正 / 定期メンテナンス / ドキュメント更新 / その他 |
| 担当者 | 作業を実施したメンバー |
| 登録日 | 作業を登録した日時 |

※ 将来的に「作業時間」が追加されることで、さらに深い分析が可能になる。

---

## 2. 分析の視点と閲覧できる情報

### 2.1 作業量の全体傾向

**目的**: 案件外作業がどの程度発生しているかの全体像を把握する

| 分析内容 | 閲覧できる情報 | 有効なグラフ |
|---------|--------------|------------|
| 週別の作業件数推移 | 案件外作業が増加傾向か減少傾向か | **折れ線グラフ** |
| 月別の作業件数推移 | 月単位での作業ボリュームの変動 | **棒グラフ** |
| 曜日別の作業件数 | 特定の曜日に集中しているか | **棒グラフ** |

**活用例**:
- 「月末に突発作業が増える」→ 月末の体制を厚くする判断材料
- 「案件外作業が毎月増えている」→ 人員増やプロセス見直しの根拠

---

### 2.2 分類別の構成比分析

**目的**: 定常保守・突発保守・依頼対応のバランスを把握する

| 分析内容 | 閲覧できる情報 | 有効なグラフ |
|---------|--------------|------------|
| 分類別の件数割合 | 3分類の構成比 | **円グラフ / ドーナツグラフ** |
| 分類別の件数推移 | 各分類が時系列でどう変化しているか | **積み上げ棒グラフ** |
| 分類別 × 作業種別のクロス集計 | どの分類にどの種別が多いか | **ヒートマップ / 積み上げ棒グラフ** |

**活用例**:
- 「突発保守が全体の50%を占めている」→ 予防保守の強化を検討
- 「依頼対応が増加傾向」→ 案件化すべき作業がないか精査

---

### 2.3 作業種別の分析

**目的**: どのような種類の作業に時間が取られているかを把握する

| 分析内容 | 閲覧できる情報 | 有効なグラフ |
|---------|--------------|------------|
| 作業種別ごとの件数ランキング | 最も多い作業種別は何か | **横棒グラフ** |
| 作業種別の月別推移 | 特定の種別が増減しているか | **折れ線グラフ（複数系列）** |
| 作業種別 × 担当者のクロス集計 | 誰がどの種別を多く担当しているか | **ヒートマップ** |

**活用例**:
- 「問い合わせ対応が最多」→ FAQ整備やセルフサービス化を検討
- 「バグ修正が特定時期に集中」→ リリース後のバグ発生パターンを分析

---

### 2.4 担当者別の分析

**目的**: メンバー間の作業負荷の偏りを可視化する

| 分析内容 | 閲覧できる情報 | 有効なグラフ |
|---------|--------------|------------|
| 担当者別の作業件数 | 誰がどれだけ案件外作業を行っているか | **横棒グラフ** |
| 担当者別 × 分類の内訳 | 各メンバーの作業の性質 | **積み上げ横棒グラフ** |
| 担当者別の週次推移 | 特定メンバーへの偏りが常態化しているか | **折れ線グラフ（複数系列）** |

**活用例**:
- 「Aさんに突発保守が集中」→ ナレッジ共有やローテーションを検討
- 「特定メンバーの作業量が突出」→ 負荷分散の判断材料

---

### 2.5 リピート作業の分析

**目的**: 繰り返し発生する作業を特定し、効率化・自動化の候補を見つける

| 分析内容 | 閲覧できる情報 | 有効なグラフ |
|---------|--------------|------------|
| 同一作業名の出現頻度 | 繰り返し登録されている作業 | **ワードクラウド / 横棒グラフ** |
| リピート作業の発生間隔 | どの頻度で繰り返されているか | **タイムライン / 散布図** |

**活用例**:
- 「毎週同じ作業が登録されている」→ 自動化・定型化の候補
- 「同じ問い合わせが繰り返される」→ ドキュメント化で削減可能

---

## 3. 作業時間の追加による拡張分析（Phase 3）

作業時間が記録できるようになると、件数ベースの分析に加え以下が可能になる。

| 分析内容 | 閲覧できる情報 | 有効なグラフ |
|---------|--------------|------------|
| 分類別の総工数 | 各分類にどれだけ時間を使っているか | **ドーナツグラフ** |
| 作業種別の平均作業時間 | 種別ごとの1件あたりコスト | **横棒グラフ** |
| 担当者別の総工数 | メンバーの時間的な負荷 | **積み上げ横棒グラフ** |
| 案件外作業の総工数 vs 稼働時間 | チーム稼働の何%が案件外か | **ゲージチャート / KPIカード** |
| 月別の工数推移 | 案件外作業の時間的コストの傾向 | **エリアグラフ** |

**活用例**:
- 「案件外作業がチーム稼働の30%を占めている」→ 経営への報告・人員計画の根拠
- 「突発保守の1件あたり平均2時間」→ 予防保守への投資判断

## 4. ダッシュボード構成案

データが蓄積された段階で、以下の構成のダッシュボードを検討する。

```
┌─────────────────────────────────────────────────┐
│  KPIカード                                        │
│  [今週の件数] [分類別内訳] [前週比 ±n件]            │
├────────────────────┬────────────────────────────┤
│  分類別構成比        │  週別作業件数推移             │
│  (ドーナツグラフ)     │  (積み上げ棒グラフ)           │
├────────────────────┼────────────────────────────┤
│  担当者別作業件数     │  作業種別ランキング           │
│  (積み上げ横棒)      │  (横棒グラフ)                │
├────────────────────┴────────────────────────────┤
│  リピート作業 TOP10                                │
│  (テーブル: 作業名 / 回数 / 最終登録日)              │
└─────────────────────────────────────────────────┘
```

## 5. 分析の段階的な進め方

| Phase | 内容 | 必要データ |
|-------|------|----------|
| Phase 1（現在） | 週次の件数確認、定例会での報告 | 作業名・分類・種別・担当者・登録日 |
| Phase 2 | 月次の傾向分析、負荷偏りの可視化 | 数ヶ月分の蓄積データ |
| Phase 3 | 工数ベースの分析、コスト可視化 | 作業時間の追加 |
| Phase 4 | ダッシュボード化、経営報告 | 上記すべて + 可視化ツール |

まずはPhase 1でデータを蓄積し、Phase 2の傾向分析ができるデータ量（2〜3ヶ月分）が溜まった段階で分析機能の実装を検討する。
