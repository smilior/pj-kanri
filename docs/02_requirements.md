# 要件定義書

## 1. 機能要件

### 1.1 作業登録

- 最小限の入力で作業を登録できる
- 入力項目は「作業名」「担当者」「分類」の3つを必須とし、それ以外は任意
- 担当者はプルダウンから選択（8名のメンバーリスト）
- 分類は「定常業務」「突発業務」から選択
- 登録すると即座に一覧に反映される

### 1.2 作業一覧表示

- 登録された作業を一覧表で表示する
- 表示項目: 作業名、担当者、分類、作業種別、ステータス、登録日
- 最新の登録が上に表示される

### 1.3 フィルタ

- 担当者別にフィルタできる
- 分類別（定常 / 突発）にフィルタできる
- ステータス別にフィルタできる

### 1.4 作業編集・削除

- 誰でも編集・削除が可能（認証なしのため制限しない）
- ステータスを変更できる（未着手 → 対応中 → 完了）

## 2. 画面構成

| 画面 | 概要 |
|------|------|
| メイン画面 | 作業一覧表 + フィルタ + 登録フォーム（1画面に集約） |

- 画面遷移なしのシングルページ構成
- 登録フォームは画面上部またはモーダルで表示

## 3. データ項目

| 項目 | 型 | 必須 | 備考 |
|------|----|------|------|
| 作業名 | テキスト | Yes | 自由入力 |
| 担当者 | 選択 | Yes | 8名のメンバーリスト |
| 分類 | 選択 | Yes | 定常業務 / 突発業務 |
| 作業種別 | 選択 | No | 問い合わせ対応、バグ修正、定期メンテナンス、ドキュメント更新、その他 |
| ステータス | 選択 | Auto | 未着手（デフォルト）→ 対応中 → 完了 |
| メモ | テキスト | No | 自由入力 |
| 登録日 | 日付 | Auto | 登録時に自動設定 |

## 4. 非機能要件

- **デバイス対応**: PC優先。レスポンシブ対応でスマホからも閲覧可能
- **データ保存**: MVPではローカルストレージを使用（サーバー不要）
- **認証**: なし。担当者はプルダウンで名前を選択するのみ
- **ブラウザ対応**: Chrome最新版

## 5. MVP対象外（将来検討）

- 作業時間の記録・集計
- ダッシュボード（担当者別の作業量グラフなど）
- ログイン認証・権限管理
- 通知機能（Slack連携など）
- データのエクスポート（CSV等）
